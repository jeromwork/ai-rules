# Goal
Inventory the project, create memory structure, collect constraints.

# Inputs
- Repository overview
- Environments & secrets policy
- Domain glossary (if any)

# Output
- Knowledge base skeleton created (folders/files)
- Risks & assumptions list
- Open questions

# Assistant Steps
1. Scan repo structure and detect modules.
2. Propose knowledge folders (architecture, api, domain, decisions, glossary).
3. List risks/assumptions and missing info.
